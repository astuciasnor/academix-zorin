Tutorial Completo de Instalação e Configuração no Zorin OS Lite

Bem-vindo ao seu guia completo para configurar rapidamente seu recém-instalado Zorin OS Lite! Este tutorial utiliza uma série de scripts automatizados para instalar seus programas favoritos e realizar configurações essenciais de forma eficiente.

Por que usar este tutorial?
  • Automação: Instale múltiplos programas com um único comando.
  • Eficiência: Economize tempo e evite erros manuais.
  • Robustez: Os scripts são projetados para lidar com dependências e problemas comuns de instalação.
  • Organização: Todos os links e instruções em um só lugar.

Para começar, certifique-se de que todos os arquivos necessários estejam na sua pasta Downloads, conforme as instruções abaixo.

Início - Arquivos Necessários e Links para Download

Antes de iniciar as instalações, você precisará baixar alguns arquivos e garantir que todos os scripts estejam no local correto.

1. Links Oficiais para Download dos Arquivos .deb e .tar.gz

Baixe os arquivos dos programas listados abaixo e coloque-os **TODOS na sua pasta ~/Downloads**. Verifique os nomes exatos no site oficial, pois podem mudar com novas versões:

  • Google Chrome (.deb) (Ex: google-chrome-stable_current_amd64.deb)
  • RStudio Desktop (.deb) - Procure pela versão "Ubuntu 22.04+" ou "Jammy" (Ex: rstudio-X.Y.Z-amd64.deb)
  • OnlyOffice (.deb) - Procure pela versão "Debian, Ubuntu" (Ex: onlyoffice-desktopeditors_amd64.deb)
  • WPS Office (.deb) - Procure pela versão para "Linux" (Ex: wps-office_11.X.X.XXXX_amd64.deb)
  • PeaZip (.deb) - Escolha a versão "GTK2" para sistemas baseados em Debian/Ubuntu (Ex: peazip_X.Y.Z.LINUX.GTK2-1_amd64.deb)
  • Quarto (.deb) - Escolha a versão "Debian/Ubuntu" (Ex: quarto-1.X.X-linux-amd64.deb)
  • Julia (Linux Binary - .tar.gz) - Na seção "Current stable release", baixe "Linux Binary (x64)" (Ex: julia-X.Y.Z-linux-x64.tar.gz)
  • Python (Página de Downloads) - **Você não precisa mais baixar o .tar.xz para o Python aqui, pois ele será instalado via PPA.** Esta página serve apenas como referência para versões.
  • Visual Studio Code (Página Principal) - **Você não precisa mais baixar o .deb para o VS Code aqui, pois ele será instalado via Repositório Oficial da Microsoft.**

2. Seus Scripts .sh

Certifique-se de que todos os scripts de instalação que você criou ou que foram fornecidos estejam **TAMBÉM na sua pasta ~/Downloads**. Isso é crucial para que o script principal possa encontrá-los e executá-los facilmente.

Os nomes esperados para os seus scripts são:

  • install_all_programs.sh (O script mestre que você executará)
  • install_chrome.sh
  • install_onlyoffice.sh
  • install_peazip.sh
  • install_pinta_geogebra_ppa.sh
  • install_flameshot_flatpak.sh
  • install_r_rstudio.sh
  • install_wps.sh
  • install_zotero_flatpak.sh
  • install_quarto.sh
  • install_julia.sh
  • install_python_vscode.sh
Importante: Se os nomes dos seus scripts forem diferentes, você precisará renomeá-los para corresponder à lista acima para que o install_all_programs.sh funcione corretamente.

Script Principal (install_all_programs.sh)

O que o install_all_programs.sh faz?

Este é o script central do seu processo de instalação. Ele foi projetado para automatizar a execução de todos os outros scripts individuais que você organizou. Ao rodar este script, ele:

  • Solicita a senha de administrador (sudo) apenas uma vez no início.
  • Chama cada um dos scripts específicos (Chrome, OnlyOffice, WPS, etc.) em uma sequência predefinida.
  • Verifica o sucesso de cada instalação individual e para o processo geral se encontrar algum erro crítico.
  • Fornece feedback no terminal sobre o progresso e a finalização de cada etapa.

Como Usar o Script Principal

Com todos os seus arquivos .deb e .tar.gz (Julia) e .sh organizados na pasta ~/Downloads, siga estas etapas:

  1. Abra o Terminal na Pasta Downloads:

    A forma mais simples é navegar até a pasta Downloads no seu gerenciador de arquivos (Ex: Nautilus, Dolphin, Thunar), clicar com o botão direito do mouse em um espaço vazio e selecionar a opção "Abrir Terminal Aqui" ou "Open Terminal Here".

    Alternativamente, você pode abrir o terminal normalmente (Ctrl + Alt + T) e digitar:

    cd ~/Downloads
  2. Dê Permissão de Execução a Todos os Scripts:

    Este passo é crucial para que os scripts possam ser executados. No terminal (já na pasta Downloads), digite:

    chmod +x *.sh

    Este comando aplica permissão de execução a *todos* os arquivos que terminam com `.sh` na sua pasta atual.

  3. Execute o Script Principal:

    Agora, você pode iniciar o processo de instalação completo. No terminal, digite:

    ./install_all_programs.sh

    O script pedirá que você pressione ENTER para continuar. Aguarde a conclusão de todas as instalações.

Estrutura e Ordem de Instalação

O script install_all_programs.sh executa os instaladores na seguinte ordem:

  1. Google Chrome (install_chrome.sh)
  2. OnlyOffice (install_onlyoffice.sh)
  3. WPS Office (install_wps.sh)
  4. PeaZip (install_peazip.sh)
  5. Pinta e GeoGebra (install_pinta_geogebra_ppa.sh)
  6. Flameshot (install_flameshot_flatpak.sh)
  7. Zotero (install_zotero_flatpak.sh)
  8. Julia (install_julia.sh)
  9. Quarto (install_quarto.sh)
  10. R e RStudio (install_r_rstudio.sh)
  11. Python (via PPA) e Visual Studio Code (via Repo Oficial) (install_python_vscode.sh)
Finalização: Ao final da execução do script mestre, será recomendado reiniciar o sistema. Isso é importante para aplicar todas as mudanças, especialmente para o WPS Office e outras configurações de ambiente.

Google Chrome

Introdução

Este script automatiza a instalação do Google Chrome no seu Zorin OS Lite a partir do arquivo .deb que você baixou. Ele garante que o Chrome seja instalado corretamente e tenta resolver automaticamente qualquer dependência ausente.

Lembre-se: O arquivo google-chrome-stable_current_amd64.deb deve estar na sua pasta ~/Downloads.

Como o script install_chrome.sh funciona

O script procura o arquivo .deb do Chrome na pasta Downloads. Ele então utiliza o comando sudo apt install -y ./, que é a maneira mais robusta de instalar pacotes locais, pois o apt automaticamente tenta resolver e instalar quaisquer dependências necessárias. Se a instalação falhar por dependências, ele executa sudo apt-get install -f -y para corrigir e tenta instalar novamente.

Execução Individual (opcional)

Caso precise instalar apenas o Chrome (sem usar o script principal), no terminal (na pasta Downloads), execute:

chmod +x install_chrome.sh
./install_chrome.sh

Finalização

Depois de instalado, você pode abrir o Google Chrome pelo menu de aplicativos do Zorin OS ou digitando no terminal:

google-chrome

Para remover o Chrome no futuro, use:

sudo apt remove google-chrome-stable

Depois, tenta instalar novamente para garantir o sucesso.

Execução Individual (opcional)

chmod +x install_onlyoffice.sh
./install_onlyoffice.sh

Após a instalação, você poderá iniciar o OnlyOffice pelo menu de aplicativos ou digitando no terminal:

onlyoffice-desktopeditors

WPS Office

Introdução

Este script automatiza a instalação do WPS Office a partir do arquivo .deb e tenta configurar a tradução para o português brasileiro (pt_BR) e o corretor ortográfico. A tradução automática de interfaces complexas no Linux pode ser desafiadora devido a variações de versão, mas este script implementa a abordagem mais comum da comunidade.

Lembre-se: O arquivo wps-office_11.X.X.XXXX_amd64.deb (ou similar) deve estar na sua pasta ~/Downloads.

Como o script install_wps.sh funciona

O script primeiro instala o WPS Office base, corrigindo dependências. Em seguida, ele tenta aplicar a tradução e o corretor ortográfico:

  1. Garante a instalação do p7zip-full (necessário para extrair o pacote de tradução `mui.7z`).
  2. Baixa o pacote de tradução `mui.7z` de um repositório comunitário no GitHub.
  3. Extrai e copia os arquivos de tradução para o diretório de interface do WPS Office.
  4. Instala os dicionários de ortografia para português do Brasil (hunspell-pt-br e hyphen-pt-br) via `apt`.

Execução Individual (opcional)

Caso precise instalar apenas o WPS Office (sem usar o script principal), no terminal (na pasta Downloads), execute:

chmod +x install_wps.sh
./install_wps.sh
IMPORTANTE: Configuração de Idioma e Corretor Ortográfico no WPS Office

A tradução automática de interfaces complexas no Linux, especialmente para softwares como o WPS Office, pode ser inconsistente devido à incompatibilidade de versões entre o programa e os pacotes de tradução externos. Se a interface ou o corretor ortográfico não estiverem em português após a instalação e um reinício, siga estas etapas:

  1. Abra qualquer aplicativo do WPS Office (Writer, Spreadsheet, Presentation).
  2. Vá em 'Tools' (Ferramentas) -> 'Switch Language' (Mudar Idioma). Se 'Português (Brasil)' estiver disponível, selecione-o e reinicie o aplicativo.
  3. **Se o problema persistir ou a opção não aparecer:**
    • Acesse as configurações de idioma/ortografia do WPS Office (geralmente em 'Tools' -> 'Options' -> 'Spell Check' ou 'Language').
    • Dentro do WPS, procure e baixe os pacotes de idioma e/ou dicionários de ortografia diretamente através da interface do aplicativo. Este é o método mais confiável.
Recomenda-se reiniciar o sistema após a instalação do WPS para garantir que todas as alterações se apliquem.

Suporte e dúvidas

Se precisar de ajuda com este script ou com a instalação do WPS Office, consulte a documentação oficial ou os recursos de suporte.

Flameshot

Introdução

Este script instala o **Flameshot**, uma poderosa ferramenta de captura de tela com recursos avançados de edição, utilizando o método de instalação **Flatpak**. Esta é a forma mais recomendada para garantir uma instalação estável e atualizada, evitando problemas com PPAs.

Vantagens do Flatpak para Flameshot:
  • **Instalação Simples:** Comando único e confiável.
  • **Atualizações Automáticas:** Gerenciado pelo Flatpak, junto com outros aplicativos.
  • **Compatibilidade:** Funciona de forma consistente em diversas distribuições.

Como o script install_flameshot_flatpak.sh funciona

O script primeiro verifica se o Flatpak está instalado no seu sistema. Se não estiver, ele o instala. Em seguida, ele garante que o repositório Flathub (o principal repositório de aplicativos Flatpak) esteja configurado. Finalmente, ele instala o Flameshot diretamente do Flathub.

Execução Individual (opcional)

Caso precise instalar apenas o Flameshot (sem usar o script principal), no terminal (na pasta Downloads), execute:

chmod +x install_flameshot_flatpak.sh
./install_flameshot_flatpak.sh

Finalização

Após a instalação, você pode iniciar o Flameshot procurando por "Flameshot" no menu de aplicativos. Para uma captura rápida, digite flameshot gui no terminal.

É **altamente recomendável** configurar um atalho de teclado (ex: tecla Print Screen) para o Flameshot nas configurações do sistema para maior conveniência:

  1. Vá em 'Configurações do Sistema' -> 'Teclado' -> 'Atalhos de teclado' (ou similar, o caminho pode variar ligeiramente no Zorin OS).
  2. Adicione um atalho personalizado.
  3. Nomeie-o como 'Flameshot' ou 'Captura de Tela'.
  4. Para o comando, insira: 'flameshot gui'.
  5. Associe a tecla Print Screen (ou outra de sua preferência) a este atalho.

Para manter seus aplicativos Flatpak (incluindo o Flameshot) atualizados, você pode executar periodicamente:

flatpak update

Julia

Introdução

Este script automatiza a instalação da linguagem de programação **Julia**, uma linguagem de alto desempenho para computação técnica e científica. A instalação é feita a partir do arquivo binário oficial (.tar.gz) para garantir a versão mais recente e estável, e a integração com o Visual Studio Code é configurada.

Lembre-se: O arquivo julia-X.Y.Z-linux-x64.tar.gz (ou similar, onde X.Y.Z é a versão) deve estar na sua pasta ~/Downloads. Baixe-o de julialang.org/downloads/.

Como o script install_julia.sh funciona

O script procura o arquivo .tar.gz do Julia na pasta Downloads. Em seguida, ele realiza os seguintes passos:

  1. Descompacta o arquivo em um diretório temporário.
  2. Move a pasta extraída do Julia para /opt/ (um local padrão para softwares instalados globalmente que não são gerenciados por pacotes do sistema).
  3. Cria um link simbólico (`symlink`) para o executável `julia` em `/usr/local/bin/`, o que permite que você execute `julia` diretamente de qualquer lugar no terminal.
  4. Limpa os arquivos temporários.
  5. Instala a extensão oficial da Julia para o Visual Studio Code, se o VS Code já estiver instalado.

Execução Individual (opcional)

Caso precise instalar apenas Julia (sem usar o script principal), no terminal (na pasta Downloads), execute:

chmod +x install_julia.sh
./install_julia.sh

Finalização

Após a instalação, você pode iniciar a REPL (Read-Eval-Print Loop) da Julia digitando julia no terminal.

Para usar Julia no Visual Studio Code, a extensão já deve ter sido instalada (se o VS Code estava presente). No VS Code, você pode abrir a Paleta de Comandos (Ctrl+Shift+P ou Cmd+Shift+P no macOS) e digitar "Julia: Start REPL" ou "Julia: New File" para começar a programar.

R e RStudio

Introdução

Este script é um instalador completo para o ambiente R no seu Zorin OS Lite. Ele instala o R e todas as dependências de sistema essenciais para a maioria dos pacotes R populares, a fonte Cascadia Code (ótima para programação) e, opcionalmente, o RStudio Desktop.

Lembre-se: Se for instalar o RStudio via script, o arquivo rstudio-*.deb deve estar na sua pasta ~/Downloads.

Como o script install_r_rstudio.sh funciona

O script executa uma série de passos robustos:

  1. Atualiza o sistema e instala ferramentas essenciais.
  2. Adiciona a chave pública e o repositório CRAN oficial do R, garantindo a versão mais recente e estável.
  3. Instala o R e suas ferramentas de desenvolvimento (`r-base-dev`).
  4. Instala uma extensa lista de bibliotecas de desenvolvimento de sistema que são pré-requisitos para compilar e usar muitos pacotes R (por exemplo, para gráficos, conexão com banco de dados, manipulação de dados).
  5. Instala a fonte Cascadia Code.
  6. Detecta e instala automaticamente o arquivo `.deb` do RStudio se encontrado na sua pasta Downloads, corrigindo dependências se necessário.

Execução Individual (opcional)

Caso precise instalar apenas R e RStudio (sem usar o script principal), no terminal (na pasta Downloads), execute:

chmod +x install_r_rstudio.sh
./install_r_rstudio.sh

Instalação Avançada do RStudio via wget (Método Alternativo)

Se você preferir sempre a versão mais recente do RStudio Desktop diretamente do site da Posit (mesmo que o script principal já tente instalar o `.deb` local), pode seguir estes passos após a instalação do R:

  1. Acesse https://posit.co/downloads/
  2. Clique com o botão direito do mouse no nome do arquivo .deb desejado e selecione “Copiar endereço do link”.
  3. No terminal, use os comandos a seguir, substituindo o link e o nome do arquivo conforme o que você copiou: 
    wget 
sudo apt install gdebi-core
sudo gdebi nome_do_arquivo_baixado.deb
    Exemplo: 
    wget https://download1.rstudio.org/electron/jammy/amd64/rstudio-2025.05.0-496-amd64.deb
sudo apt install gdebi-core
sudo gdebi rstudio-2025.05.0-496-amd64.deb

Este método funciona para qualquer versão futura do RStudio diretamente do site oficial!

Dica de Instalação de Pacotes no R

install.packages(c("tidyverse", "data.table", "devtools", "shiny", "remotes", "readxl", "openxlsx", "BiocManager"))

Para instalar pacotes do Bioconductor:

BiocManager::install("limma")

Instalação e Configuração do Python e VS Code

Introdução

Este tutorial utiliza o script install_python_vscode.sh para configurar um ambiente de desenvolvimento Python moderno e eficiente no seu Zorin OS Lite. Ele instala uma versão específica do Python (via PPA deadsnakes) e o Visual Studio Code (VS Code via repositório oficial da Microsoft), além de preparar um projeto inicial com bibliotecas essenciais.

Vantagens deste método:
  • **Simplicidade:** Não exige mais downloads de arquivos `.tar.xz` ou `.deb` para Python e VS Code; tudo é gerenciado pelo `apt`.
  • **Atualizações Fáceis:** Python e VS Code serão atualizados com o restante do sistema via `sudo apt update && sudo apt upgrade`.
  • **Consistência:** Instalação confiável em sistemas baseados em Ubuntu.
Observação sobre a versão do Python: O PPA deadsnakes oferece as versões estáveis mais recentes do Python (ex: 3.10, 3.11, 3.12). Python 3.13.4 é uma versão de desenvolvimento e pode não estar disponível neste PPA ainda. Se você *precisa* da 3.13.4 especificamente, a compilação manual (com a correção do `tk-dev`) é a única forma, mas é menos robusta e consistente. O script usará a última versão estável especificada (3.12 no exemplo).

Como o script install_python_vscode.sh funciona

O script realiza os seguintes passos de forma automatizada:

  1. **Python:**
    • Adiciona o PPA `deadsnakes` (repositório que fornece versões mais novas do Python).

  3. **Ambiente de Desenvolvimento Python:**
    • Cria um diretório de projeto exemplo (~/PythonProjects/my_first_project).
    • Cria um ambiente virtual (`.venv`) isolado dentro do projeto, usando a versão do Python instalada via `deadsnakes`.

    • Gera arquivos `main.py` (com código de exemplo) e `README.md` (com instruções de uso) no projeto.

Configuração do GRUB

Introdução

O GRUB (Grand Unified Bootloader) é o gerenciador de boot do seu sistema Linux. Se você tem um dual boot com Windows (ou outro sistema operacional), configurar o GRUB permite definir qual sistema inicia por padrão e por quanto tempo o menu de seleção fica visível. Este tutorial vai te ajudar a definir o Windows como opção padrão no boot.

Passo 1: Listar entradas do GRUB para identificar o Windows

Abra o terminal e execute o comando para listar as entradas do menu do GRUB. Isso é importante para identificar o nome exato da sua entrada do Windows:

grep "menuentry '" /boot/grub/grub.cfg

Você verá uma lista parecida com esta. **Anote exatamente** o nome da entrada do Windows que aparece entre aspas simples, por exemplo:

menuentry 'Zorin' --class zorin ... {
menuentry 'Windows Boot Manager (on /dev/nvme0n1p1)' --class windows --class os ... {
menuentry 'UEFI Firmware Settings' ... {
'Windows Boot Manager (on /dev/nvme0n1p1)'

Passo 2: Editar o arquivo de configuração principal do GRUB

Agora, abra o arquivo de configuração para edição. Use nano (um editor de texto simples no terminal):

sudo nano /etc/default/grub

Localize a linha que começa com GRUB_DEFAULT= e substitua pelo nome completo da entrada do Windows que você anotou no Passo 1. Use aspas duplas:

GRUB_DEFAULT="Windows Boot Manager (on /dev/nvme0n1p1)"

Certifique-se de que o texto dentro das aspas seja idêntico ao que você anotou!

Passo 3: Configurar tempo de espera e exibição do menu

No mesmo arquivo (`/etc/default/grub`), ajuste as linhas abaixo para definir o tempo que o GRUB exibirá o menu antes de iniciar o sistema padrão automaticamente:

GRUB_TIMEOUT=10
GRUB_TIMEOUT_STYLE=menu

Onde:

  • GRUB_TIMEOUT=10: O menu ficará 10 segundos visível. Você pode mudar para qualquer valor em segundos.
  • GRUB_TIMEOUT_STYLE=menu: Garante que o menu seja exibido durante o tempo configurado.

Se quiser que o menu espere indefinidamente até que o usuário selecione a opção e pressione Enter, defina:

GRUB_TIMEOUT=-1

Passo 4: Salvar e atualizar o GRUB

Após fazer as edições no arquivo, salve e feche o editor (no `nano`: pressione Ctrl+O para salvar, depois Enter para confirmar o nome do arquivo, e Ctrl+X para sair).

Em seguida, rode o comando para que o GRUB aplique as novas configurações:

sudo update-grub

Passo 5: Reiniciar e verificar a configuração

Por fim, reinicie o computador para ver as mudanças em ação:

sudo reboot

Na tela do GRUB, o Windows deve aparecer como a primeira opção pré-selecionada, aguardando o tempo configurado para iniciar automaticamente, ou aguardando o Enter se você configurou o `GRUB_TIMEOUT=-1`.

Dica extra para ambientes de laboratório

Para evitar boot automático e garantir que o operador sempre escolha manualmente o sistema, especialmente em ambientes onde múltiplos sistemas são usados ou onde a atenção é necessária para a inicialização:

GRUB_TIMEOUT=-1

Assim, o menu ficará visível até que alguém pressione Enter para iniciar o sistema selecionado.
