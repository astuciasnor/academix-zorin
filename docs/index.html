Primeiros Passos Pós-Instalação

O processo de instalação automatizado cuidou de tudo. O recomendado agora é reiniciar seu computador para garantir que todas as configurações sejam aplicadas. Após reiniciar, você encontrará todos os novos programas no menu de aplicativos do Zorin OS.

Atualizações Futuras: Para manter os aplicativos instalados via Flatpak (Zotero, Flameshot, Pinta) atualizados, você pode rodar periodicamente o seguinte comando no terminal: 
flatpak update
Os outros programas serão atualizados junto com o sistema através do "Atualizador de Programas" do Zorin.

WPS Office & Zotero

Sua suíte de escritório principal, WPS Office, foi instalada com a tradução para Português (Brasil). A integração com o Zotero também foi configurada. Para começar a citar seus trabalhos, abra o WPS Writer e procure pela nova aba "Zotero" na barra de ferramentas.

Use o Okular para visualizar PDFs e outros formatos de documento. Para fazer anotações, rabiscos e destaques em artigos e slides, o Xournal++ é a ferramenta perfeita.

Módulo: Desenvolvimento Científico

R, RStudio & Quarto

Seu ambiente para análise de dados está pronto. O R foi instalado a partir do repositório CRAN oficial, e o RStudio é a versão mais recente compatível. O Quarto, para publicação científica, já está integrado ao RStudio.

Python & VS Code

O Visual Studio Code foi instalado e configurado para desenvolvimento em Python. Uma versão recente do Python (via PPA `deadsnakes`) está disponível. Para criar ambientes virtuais isolados, use o comando: python3.12 -m venv nome_do_ambiente.

Inkscape

Para criar figuras, diagramas e gráficos vetoriais com qualidade de publicação, utilize o Inkscape. É a alternativa de código aberto padrão da indústria ao Adobe Illustrator.

Guia Aprimorado: Configurando o GRUB (Dual Boot)

Introdução

Se você utiliza o Zorin OS em dual boot com o Windows, pode querer que o Windows seja a opção padrão na inicialização. Este guia mostra como fazer isso de forma segura.

Atenção: A edição de arquivos de boot é uma operação sensível. Siga os passos com atenção. Um erro de digitação pode afetar a inicialização do sistema. Faça por sua conta e risco.

Passo 1: Identificar a Entrada do Windows

Primeiro, precisamos saber o nome exato que o GRUB usa para o Windows. Abra o terminal e execute:

grep "menuentry '" /boot/grub/grub.cfg

A saída listará todas as entradas do menu. Procure pela linha do Windows e **copie o texto exato que está entre as aspas simples**. Por exemplo:

'Windows Boot Manager (on /dev/nvme0n1p1)'

Passo 2: Editar o Arquivo de Configuração do GRUB

Agora, vamos editar o arquivo de configuração principal do GRUB com um editor de texto no terminal:

sudo nano /etc/default/grub

Dentro do editor, localize a linha que diz GRUB_DEFAULT=0. Apague o 0 e cole o texto que você copiou, agora entre aspas duplas. A linha deve ficar assim:

GRUB_DEFAULT="Windows Boot Manager (on /dev/nvme0n1p1)"

Passo 3: Ajustar o Tempo de Espera (Timeout)

No mesmo arquivo, você pode alterar o tempo que o menu fica visível. Encontre a linha GRUB_TIMEOUT=10 e altere o valor `10` para o número de segundos que desejar (ex: `5` para 5 segundos).

Para que o menu sempre espere sua escolha (não inicie automaticamente), mude para GRUB_TIMEOUT=-1.

Passo 4: Salvar e Aplicar as Mudanças

Para salvar o arquivo no editor `nano`, pressione `Ctrl+O`, depois `Enter`, e então `Ctrl+X` para sair.

De volta ao terminal, execute o comando final para que o sistema aplique suas novas configurações:

sudo update-grub

Passo 5: Reiniciar

Pronto! Na próxima vez que reiniciar o computador, o Windows estará pré-selecionado no menu do GRUB.
